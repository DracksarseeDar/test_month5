Шаг 1: Создай файл .gitignore (Критически важно!)
В корне твоего проекта (там же, где лежат manage.py и README.md) создай файл с названием строго .gitignore (с точкой в начале).
Открой его и вставь этот текст:


# Виртуальное окружение
venv/
env/

# Секретные ключи и настройки
.env

# Локальная база данных (если осталась от старых тестов)
db.sqlite3

# Кэш Питона (мусорные файлы)
__pycache__/
*.pyc
*.pyo
Это скажет Гиту: "не смотри на эти файлы и не копируй их".

Шаг 2: Сохрани код локально (в терминале)
Открой терминал в папке своего проекта (~/Desktop/moth_5/test/test_work) и введи команды по очереди:

Инициализируем Git (создаем локальный репозиторий):


git init
Добавляем все файлы (кроме тех, что в .gitignore):


git add .
Делаем коммит (сохраняем текущее состояние с комментарием):


git commit -m "Initial commit: Blog API completed"
Шаг 3: Создай пустой репозиторий на GitHub
Зайди на GitHub под своим аккаунтом.

Нажми зеленую кнопку New (создать новый репозиторий).

Напиши название, например django-blog-api.

Важно: Никакие галочки внизу (Add a README file, Add .gitignore) НЕ ставь, потому что эти файлы у нас уже есть!

Нажми кнопку Create repository.

Шаг 4: Отправь код в интернет
Сразу после создания GitHub покажет тебе страницу с подсказками. Найди там блок, который называется "…or push an existing repository from the command line".

Скопируй оттуда три строчки и по очереди вставь в свой терминал. Обычно они выглядят так:


# Подключаем твою локальную папку к серверу GitHub (ссылка будет твоя)
git remote add origin https://github.com/ТВОЙ_НИК/django-blog-api.git

# Переименовываем главную ветку в main
git branch -M main

# Отправляем файлы!
git push -u origin main
(При команде push терминал может попросить тебя ввести логин и пароль/токен от GitHub).

Готово! Обнови страницу своего репозитория в браузере, и ты увидишь там весь свой код и красивую инструкцию из README.md, которую мы сделали.
